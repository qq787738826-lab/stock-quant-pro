# CHANGELOG

## 1.3.1 - 2026-07-13

- 修复历史扫描任务接口有数据但智能选股表格为空的问题。
- 后端统一解析 PostgreSQL `jsonb` 字段，避免返回 `PGobject` 包装结构。
- 前端增加 JSON 字段兼容处理，不再直接对对象调用 `join()`。
- 历史任务自动关闭“只看严格候选”，并明确标记为历史结果。
- 从旧任务 `metrics` 中回填5日、20日、RSI、ATR和量比指标。

## 1.3.0 - 2026-07-13

- 智能选股候选新增“一键加入交易计划”。
- 交易计划支持自动计算建议数量并生成待确认买入委托。
- 待确认买入冻结模拟资金，待确认卖出冻结可卖数量。
- 人工确认成交后自动更新账户现金、持仓成本、可卖数量和盈亏。
- 新增模拟成交流水、佣金、印花税、过户费和已实现盈亏记录。
- 新增单股仓位、持仓数量、可用资金和重复委托风控。
- 新增T+1可卖数量模拟。
- 新增止损、目标价和移动止盈风险事件。
- 新增每日16:30持仓风险检查任务。
- 新增账户权益快照和权益曲线。
- 新增Flyway迁移 `V4__simulated_trading.sql`。

## 1.2.2 - 2026-07-13

- 扫描任务与结果按 `taskId` 绑定，默认展示最近一次正式全市场扫描。
- 新增扫描历史、扫描类型、正式结果标记、失败明细和失败股票重试。
- 新增行情增量更新任务，按最近90日数据进行主键覆盖入库。
- 新增严格候选过滤：评分、风险、成交额、ATR、20日涨幅、回撤、量比和涨停风险。
- 智能选股页面增加任务切换、严格候选开关、核心指标和过滤状态。
- 新增候选前N名批量历史回测，展示平均收益、平均胜率、平均回撤和个股明细。
- 数据管理中心增加扫描历史、行情更新进度和失败记录。

## 1.2.1 - 2026-07-13

- 修复数据管理中心显示 `Network Error` 的本地连接问题。
- Vite代理目标由 `localhost:8080` 改为 `127.0.0.1:8080`。
- Electron开发地址统一改为 `127.0.0.1:5173`。
- Java CORS同时允许 `localhost:5173` 和 `127.0.0.1:5173`。
- 数据管理页面增加明确的Java后端未启动提示。
- 健康检查脚本统一使用IPv4回环地址。
